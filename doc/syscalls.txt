# 通用

 00 probe_min   在控制台打印寄存器内容（r0-r7）

 01 probe       在控制台打印寄存器内容（r0-r15, flags）

 02 probe_float 在控制台打印寄存器内容（s0-s31）

 03 probe_mem   在控制台打印内存内容
      r0-r15 中所有处于有效内存地址范围内的值均视为指针，显示其前后各 16 字节

 0e log         在控制台打印一个以字符 \0 结尾的字符串
    in  r0      字符串起始地址

 0f debug       断点
      模拟器：暂停执行，控制台按下 Enter 后继续。后续可以增加一些调试指令
      设备：无作用

 10 time        自启动以来的时间，以微秒为单位
    out r0      结果的低 32 位
    out r1      结果的高 32 位

 11 key         查询键盘按键是否被按下
    in  r0      按键码
    in  r1      按键码
    in  r2      按键码
    in  r3      按键码
    out r0      0/1 表示未按下/按下
    out r1      0/1 表示未按下/按下
    out r2      0/1 表示未按下/按下
    out r3      0/1 表示未按下/按下

 12 rand        随机数
    out r0      32 位随机数
    out r1      32 位随机数
    out r2      32 位随机数
    out r3      32 位随机数

 20 pers        读写持久数据
    in  r0      数据起始地址
    in  r1      0 表示读，1 表示写
      持久数据的大小为 64 KiB。

# 图形

100 clear_frame 清除屏幕
    in r0       填充色
      寄存器中传递的颜色都是 RGBA8 格式，即从高到低分别为红、绿、蓝、alpha，各占
    8 位。如 R=0xff，G=0xee，B=0xdd，A=0xcc，则表示为 0xffeeddcc。
      清除屏幕时忽略 alpha 通道，均视为不透明色。

10f end_frame   结束当前帧
      当绘图结束时返回

110 tex_alloc   创建纹理
    in  r0      宽，以像素为单位
    in  r1      高，以像素为单位
    out r0      纹理 ID

111 tex_image   更新纹理图像
    in  r0      纹理 ID
    in  r1      原始图像数据起始地址
      原始图像数据为 RGBA8 格式，每个像素占 4 字节，依次为红、绿、蓝、alpha。像
    素按行优先顺序排列。总大小为 (长×宽×4) 字节。
      调用完成后，数据内存中的内容不必保留。

112 tex_config  设置纹理（暂时不用）
    in  r0      纹理 ID
    in  r1      设置标志位

11f tex_release 释放纹理
    in  r0      纹理 ID

120 draw        绘制三角形
    in  r0      顶点 A 颜色
    in  r1      顶点 B 颜色
    in  r2      顶点 C 颜色
    in  r3      纹理 ID，-1 表示不使用纹理
    in  s0      顶点 A 的 X（横坐标）
    in  s1      顶点 A 的 Y（纵坐标）
    in  s2      顶点 A 的 U（纹理横坐标），不使用纹理时忽略
    in  s3      顶点 A 的 V（纹理纵坐标），不使用纹理时忽略
    in  s4      顶点 B 的 X（横坐标）
    in  s5      顶点 B 的 Y（纵坐标）
    in  s6      顶点 B 的 U（纹理横坐标），不使用纹理时忽略
    in  s7      顶点 B 的 V（纹理纵坐标），不使用纹理时忽略
    in  s8      顶点 C 的 X（横坐标）
    in  s9      顶点 C 的 Y（纵坐标）
    in  s10     顶点 C 的 U（纹理横坐标），不使用纹理时忽略
    in  s11     顶点 C 的 V（纹理纵坐标），不使用纹理时忽略

# 声音

200 snd_alloc   创建声音
    in  r0      长度，以采样为单位
    out r0      声音 ID
      音频采样率为 44.1 kHz，创建时数据全为零。

201 snd_pcm     更新音频数据
    in  r0      声音 ID
    in  r1      采样数据起始地址
      音频数据均为 16 位有符号整形交错双声道采样（16-bit signed interleaved
    stereo PCM）。
      播放过程中可以更新音频数据，下一帧起即按新数据播放。
      调用完成后，数据内存中的内容不必保留。

20f snd_release 释放声音
    in  r0      声音 ID

210 play        播放声音
    in  r0      声音 ID
    in  r1      轨道编号
    in  r2      偏移，以采样为单位
    in  r3      0/1 表示是否循环
      一共设 16 个轨道，编号 0 至 15，每个轨道只能同时播放一个声音，所有轨道的声
    音经叠加混合后输出。一个声音可以在多个轨道播放多次。
      如果声音 ID 为 -1，则停止轨道上正在播放的声音，此时忽略后两个参数；同时，
    若轨道编号也为 -1，则停止所有轨道。不过各轨道的音量和声像设置都不改变。

211 trk_config  设置轨道
    in  r0      轨道编号
    in  r1      音量
    in  r2      声像
      音量：线性增益。1 表示静音，0x80000000 表示原音量，0xffffffff 表示 2 倍增
    益（+6 dB）。另外，0 也表示原音量。
      声像：1 表示最左，0x80000000 表示居中，0xffffffff 表示最右。另外，0 也表示
    居中。
      初始默认值为原音量、声像居中。

212 trk_tell    检查轨道播放状态
    in  r0      轨道编号
    out r0      当前播放的声音 ID
    out r1      当前偏移，以采样为单位
      若未播放，则声音 ID 为 -1，偏移值为 0。
