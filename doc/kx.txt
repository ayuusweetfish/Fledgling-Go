# 图像
    图像文件只支持 PNG。

kx_image          从图像文件创建纹理
  in  r0          文件数据起始地址
  in  r1          文件数据长度
  out r0          纹理 ID
  out r1          宽，以像素为单位
  out r2          高，以像素为单位

# 音频
    由于原始的音频数据占据内存非常大，kx 封装了一组函数，用于从音频文件中不断解
  码出小段的音频，减少内存占用，并在两条轨道上交替播放，保持声音无缝衔接。这样的
  一条音频由一个「音频流」表示，拥有一个惟一的 ID，这个 ID 只在 kx_music_* 函数
  中有意义。

kx_music          从音频文件创建音频流
  in  r0          文件数据起始地址
  in  r1          文件数据长度
  in  r2          第一条轨道编号
  in  r3          第二条轨道编号
  out r0          音频流 ID

kx_music_start    开始/继续播放音频流
  in  r0          音频流 ID

kx_music_pause    暂停音频流
  in  r0          音频流 ID

kx_music_update   更新音频流
  in  r0          音频流 ID
    需要每帧调用一次此函数，它会在恰当的时机解码新的音频小段。

kx_music_release  释放音频流
  in  r0          音频流 ID

# 文字（标签）
    一个「标签」包含一个 TrueType 字体和一段文本，可以用此字体显示之。文本可以随
  时重新设置，但是出于性能因素考虑，不建议更新过于频繁。

kx_label          从字体文件创建标签
  in  r0          文件数据起始地址
  out r0          标签 ID
    标签创建时文本为空。

kx_label_print    更新标签的文本
  in  r0          标签 ID
  in  r1          文本，以字符 \0 结尾
  in  s0          字号

kx_label_draw     绘制标签
  in  r0          标签 ID
  in  r1          颜色（RGBA8）
  in  s0          文本左上角的屏幕 X 坐标
  in  s1          文本左上角的屏幕 Y 坐标
  in  s2          水平缩放比例（1/屏幕宽 对应原始实际大小）
  in  s3          垂直缩放比例（1/屏幕高 对应原始实际大小）

kx_label_release  释放标签
  in  r0          标签 ID
